# Behavioral Validation

This module extracts item-level behavioral signals from the raw Hugging Face
Qilin dataset and checks whether those signals can be matched back to the local
processed Step4 samples.

The focus is on behavioral fields that were not preserved in the current
`Data/Step4/*.pkl` files:

- `page_time`
- `position`
- item-level interaction flags such as `click`, `like`, `collect`, `comment`,
  and `share`

## Data Grain

`raw_behavior_events.parquet` is item/exposure-level. It includes clicked and
unclicked items from:

- `search_train`
- `search_test`
- `recommendation_train`
- `recommendation_test`

`clicked_behavior_samples.parquet` keeps only `click == 1` rows. This is the
table intended for matching against `Data/Step4/src_all.pkl` and
`Data/Step4/rec_all.pkl`, which are clicked positive-item sample tables.

The default matching key is:

```text
channel + user_id + item_id + timestamp
```

`search_session_id` is not used as a required key because the local processed
session ids are generated by the preprocessing pipeline and are not guaranteed
to be identical to Hugging Face `session_idx`.

## Behavioral Preference Elaboration

The plotting script aligns `final_matched_samples_with_step4.parquet` with the
sample-level PC-SAR mechanism export:

```text
5-Validation/Features/intermediate/pcsar_intent_features_all_full_mechanism.csv
```

It bins `global_posterior_uncertainty` into Low, Medium, and High preference
elaboration states. Low is the bottom 20%, High is the top 20%, and Medium is
the middle 60%. The figures compare Low and High distributions separately for
recommendation and search clicks in a 2-by-2 layout:

```text
preference_elaboration_future_query_reformulation.png
preference_elaboration_dwell_time.png
preference_elaboration_click_position_depth.png
```

Future query reformulation is measured as the consecutive reformulation length
over the next at most 10 search sessions, where adjacent query-session embedding
similarity of at least 0.5 continues the run.

The dwell-time histogram excludes zero or negative `page_time` rows for display.
